## 〇、系列编排层（选题之上）

单条之上有一层"系列"：`选题清单/系列规划.md` 管"这个月讲什么大主题、分几集、按什么顺序出"。选题 skill 出题前先看它——**在排系列的下一集优先**，零散爆点单条次之。集与集要钩（上期X今天Y+续集钩子），系列全发完做"系列复盘"。
> 来源参考：XHSSpec 的 Campaign 工作流。

## 一、完整链路（👤=人工卡点，其余自动）

```
系列规划(在排系列下一集) + 热点(aihot)+数据(analytics)+套路库
   → 选题评分卡出清单 → 👤拍板选题
   → 写文案初稿 → 👤改腔调+过红线（定稿）
   → 出发布包(标题/tag/封面/预测卡) → 预测预警则👤微调
   → 录口播 → remotion出片 → 👤定稿
   → 发布 → 数据回流 → 复盘卡 → 👤确认后更新套路库&预测器
```

**为什么留这几个👤**：选题(值不值得做)、文案(AI腔+红线)、改库(防过拟合噪声)——都是 AI 的短板，必须人兜底。

---

## 二、文案只存定稿

草稿在对话里多轮碰撞，**不每版存文件**；只有用户说"定稿"才写入 `文案/`，一个文案一个文件、原地覆盖、状态从「草稿」翻「定稿」。

---

## 三、发布包 5 件套（每条定稿文案必带）

1. **口播稿**（主体）
2. **标题**：2-3 选，对应不同钩子；标题↔开头钩子必须呼应
3. **Tag**：定位标签 + 活动扶持标签（前3个权重最高）；🚫无变美、无割韭菜词
4. **封面成品**（不只是建议）：定稿风格 **Swiss IKB 宝蓝 + 真人脸**，一键出——
   `python3 视频生产/scripts/make_cover.py --title "主标题/换行" --hl "末词" --out <素材目录>/封面.png`。
   **封面和这条片的图卡一起出**（视频生产环节同步产出，缺一不可）。
5. **预测卡**（见下）

## 四、预测卡（发布前·闭环前半截）

**以比率为主、播放为辅**（北极星不看播放）：

| 指标 | 预测 | 依据 | 置信 |
|---|---|---|---|
| 收藏率(藏/赞) | … | 对标同类 | |
| 完播率 | … | 时长/类型 | |
| 涨粉/千播 | … | 受众匹配 | |
| 播放区间 | … | 账号基线+权重 | |
| 点赞区间 | … | 跟播放走 | |

⚠️ 起号期预测必不准（无自身样本，只能借对标）。**价值不在猜中，在对账迭代**。

## 五、复盘卡（发布后·闭环后半截）

每条视频发布后 24h/72h/7天 取数，产出复盘卡存 `复盘/`：

```bash
python3 scripts/loop_review.py --latest                  # 复盘最新一条
python3 scripts/loop_review.py --item-id <item_id_plain> # 复盘指定视频
# 复用 douyin-analytics 的 cookie(~/.douyin_cookie) 与取数函数；cookie 失效用 update_cookie.sh 刷新
```
脚本自动取数 → 北极星判定(命中/平庸/扑) → 留存归因 → 限流预警 → 填好复盘卡。结构：

```
预测（评分卡几分/用了哪些套路/预测卡数据）
→ 实际（北极星指标：完播/收藏率/涨粉千播/推荐流占比）
→ 判定（命中/平庸/扑，看北极星不看播放）
→ 归因（用留存曲线分离"选题问题"vs"执行问题"：开头掉=钩子，中段掉=结构）
→ 更新动作（套路库加权/降权 + 预测器校准，待👤确认）
```

## 六、闭环迭代：两条腿

- **套路库**（迭代"什么会爆"）：命中套路加权、扑的降权、新规律入库、评分卡权重调整
- **预测器**（迭代"会爆到什么程度"）：每条 prediction vs actual 的 diff，让预测从"借对标"变"贴自己账号"

**铁律**：起号期样本少、噪声大——单条扑了别急改库，等同类 2-3 条再下结论（防过拟合）。数据 24-72h 才稳，别发完就判。

## 七、对标监控（半自动）

抖音反爬挡死"列博主视频列表"，所以列表靠你贴链接，其余自动：

```bash
python3 scripts/monitor.py --links-file /tmp/new.txt   # 一批对标新视频链接
python3 scripts/monitor.py "<链接>" --min-likes 8000   # 直接传/自定阈值
```
自动：去重(ledger台账) → 抓数据 → 按点赞阈值筛爆款 → 写 `对标监控/待拆解-<日期>.md`。
深耕对标(秋芝/珍妮/柱子哥)见 `对标监控/watchlist.json`。爆款再走拆解→入套路库，喂给「选题」。
