## 1. 已敲定的工作流（重要决策）

**录后设计为主，录前只对内容、不锁帧。**

| 阶段 | 用户给 | agent 做 |
|---|---|---|
| 录前（内容层） | 选题 + 文案骨架 + 想强调/想展示的点 | 打磨 hook/结构/节奏，给松散提纲（不锁帧） |
| 录后（剪辑层） | 录完的 `.MOV` | 转写真实语音 → 逐帧设计 timeline → 补图清单 → 渲染 |

**为什么逐帧只能录后做**：字幕逐词卡点依赖词级时间戳，而时间戳只能来自真实录音。录前无法知道某字落第几帧。

**两阶段推进**：
- **阶段一（已完成搭建）**：搭通管线，借用旧字幕风格，先证明「口播→成片」跑得通。
- **阶段二（待开始）**：优化模板风格（配色/字体/强调动效/B-roll 转场/字幕版面）。数据驱动，**改风格只动 Remotion 组件，不重跑转写**。

## 3. 5 步 loop（命令）

```bash

# ⑥ 定稿后才渲染导出

```

## 4. timeline.json 契约（大脑↔渲染器，时间单位=帧）

```jsonc
{
  "fps": 30, "width": 1080, "height": 1920,
  "source": "input.mov",            // 相对项目目录，底层真人画面
  "durationInFrames": 6516,         // = 视频时长×fps；render.mjs 会自动补
  "captions": [
    { "start": 0, "end": 44, "tag": "可选黄胶囊标签",
      "words": [ { "t": "全网", "at": 0 },
                 { "t": "AI", "at": 8, "highlight": "yellow", "emoji": "✨", "punch": true } ] }
  ],
  "broll": [
    { "from": 1313, "to": 1453, "asset": null, "label": "差skill文件结构…" }  // asset=null→占位卡
  ]
}
```
- `highlight` 取值：`green|yellow|red|blue`
- `at` = 相对本屏 start 的入场帧；`start/end` 相邻屏首尾相接（字幕不闪断）

## 5. 关键技术坑（务必知道）

1. **视频资产必须是真文件/硬链，不能用符号链接**：Remotion 复制 `--public-dir` 时不跟符号链。
   `input.mov` 已用 `ln`（硬链，同卷无副本）。命令：`npx remotion render TalkingHead out.mp4 --props=<timeline.json> --public-dir=<项目目录>`，资产经 `staticFile` 从 public-dir 解析。
2. **`remotion still` 不跑 calculateMetadata**，会用 Composition 静态 duration(90) 报 frame 越界。
   验证单帧请用 `remotion render --frames=a-b` 再 ffmpeg 抽帧。
3. **转写错字**：whisper small 中文有同音错字。已在 `build_timeline.py` 的 `CONTEXT_FIX`/`ASCII_FIX` 校正：
   适→试、万份→万粉、堵定→笃定、报款→爆款、邦尼高校→帮你高效、听天有命→听天由命、
   卡斯克→卡兹克、信息园→信息源、证文→正文、国脚部→裹脚布；pront→prompt、clod→Claude。新口播需补充。
4. **B-roll 语义层放 `broll.json`**（agent 手写），build_timeline 会并进 timeline，重跑分词不会清掉。
