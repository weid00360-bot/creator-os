# 口播 → 自动剪辑成片 工作流

> 目标：给一个真人口播 `.MOV`，自动产出带逐词字幕、强调高亮、B-roll 全屏切的成片。
> 输入只支持**真人出镜竖屏视频**；口播原样保留（不剪废话/静音）。

## 分工：录前对内容，录后逐帧设计

| 阶段 | 你给 | 我做 |
|---|---|---|
| 录前（内容层） | 选题 + 文案骨架 + 想强调/想展示的点 | 打磨 hook/结构/节奏（**不锁帧**，给松散提纲） |
| 录后（剪辑层） | 录完的 `.MOV` | 转写真实语音 → 逐帧设计 `timeline.json` → 补图清单 → 渲染 |

逐帧决策（字幕卡点、高亮、B-roll 在第几帧切）**只能录后做** —— 因为词级时间戳来自你真实的录音。

## 5 步 loop

```
projects/<项目名>/
【录前·规划】（开录之前完成）
A. 你给录前文案
B. [Claude] 按 `模板库/编排方案模板.md` 逐段规划：展示方式/模板风格/素材分工/全程常驻 → 你逐段敲定
C. [Claude] 做 1-2 张关键 PPT 示意图 + 给你「拍摄+素材清单」 → 你确认风格

【录制】
D. 你按清单录口播，顺手备好要给我的录屏/截图

【录后·剪辑】
1. 你丢 input.mov + 素材
2. [脚本] transcribe.py → 词级时间戳
3. [Claude] 按敲定方案搭：timeline(逐帧字幕) + 图形/动效(按模板库风格) + 你的素材
4. [网页端] studio 预览定稿  ← 改到满意
5. [脚本] render.mjs → out.mp4（你说"导出"才跑）
6. [Claude] 复盘 → projects/<名>/复盘.md，并报告沉淀了哪些 / 模板库新增哪些
```
铁律：B 先对编排方案、4 先网页定稿；图形按 `模板库/` 选风格；6 每片必做复盘。
为什么顺：录前就定好"哪段什么形式+谁出素材"→ 你录时知道哪里留空给画面、并提前备好录屏截图 → 录后我只做时间相关的逐帧活+组装，极少返工。

> ⚠️ **铁律：看完视频先出「编排方案」跟用户核对，确认后才动手搭。** 不直接埋头干——
> 方向错了搭出来就是白干。方案要覆盖：结构理解、字幕策略、B-roll/录屏插播点、转场/特效/动效、人物+屏幕布局、背景风格。

> ⚠️ **铁律：先在 Studio 网页端预览定稿，确认后才渲染导出。**
> 全片 CPU 渲染要 10–20 分钟，每改一处就重渲是浪费；Studio 实时预览秒级看效果。
> 改完 timeline.json 或组件 → 去 Studio 看 → 满意了才 `render.mjs`。**没说"导出"就别渲染。**

## timeline.json —— 大脑↔渲染器的契约

所有时间字段单位是**帧**（秒 × fps）。

```jsonc
{
  "fps": 30,
  "width": 1080,
  "height": 1920,
  "source": "input.mov",          // 相对项目目录；渲染时作为底层真人画面
  "durationInFrames": 6516,       // 可省略，render.mjs 会按视频时长自动补全
  "captions": [
    {
      "start": 0, "end": 64,      // 本屏字幕的起止帧
      "tag": "概念扫盲",           // 可选：顶部黄色胶囊标签
      "words": [
        { "t": "全网的", "at": 0 },                                   // at = 相对本屏 start 的入场帧
        { "t": "AI博主", "at": 8, "highlight": "yellow",
          "emoji": "✨", "punch": true }                              // highlight: green|yellow|red|blue
      ]
    }
  ],
  "broll": [
    {
      "from": 120, "to": 210,                  // 全屏盖住人脸的帧区间
      "asset": null,                           // 补图前留 null → 渲染显示「待补图」占位卡
      "label": "需要：codex 跑通自媒体运营的终端截图"   // 补图清单就来自这里；补好后 asset 填 "broll/xxx.png"
    }
  ]
}
```

字段来源：
- `at / start / end / from / to` ← Whisper 词级时间戳换算成帧
- `highlight / emoji / punch / tag / broll` ← Claude 读转写后标注

## 风格（阶段二再调）

当前借用旧 `ai-video/FullVideo.jsx` 的逐词字幕样式（深色描边、霓虹高亮、spring 弹入、emoji 顶标）。
配色、字体、强调动效、B-roll 转场、字幕避开人脸的版面，都放到阶段二专门打磨。
因为是数据驱动，**改风格只动 `remotion/src/` 组件，不用重跑转写**。

## 命令速查

```bash
cd 视频生产
python3 scripts/transcribe.py projects/练手01/input.mov projects/练手01   # 转写
node   scripts/render.mjs   projects/练手01                              # 渲染
cd remotion && npm run studio                                            # 可视化调试(可选)
```
