# 发布后复盘（闭环后半截）

## 流程
1. **取数**：
   ```bash
   python3 scripts/loop_review.py --latest                  # 最新一条
   python3 scripts/loop_review.py --item-id <item_id_plain> # 指定
   ```
   复用 douyin-analytics 的 cookie(`~/.douyin_cookie`)；失效用其 `update_cookie.sh` 刷新。
   产出复盘卡 + json 到 `复盘/`。
2. **解读**：北极星判定（命中/平庸/扑，看涨粉千播+完播+收藏率，**不看播放**）。
3. **限流优先排查**：若复盘卡标 🚨疑似限流（0播放/无推荐流）→ **先查可见性/标签/红线，别归因内容**（否则把"被限流"误判成"内容不行"，污染套路库）。
4. **归因**：用留存曲线分离——开头(前5s)掉得比同类狠=钩子/选题问题；中段掉=结构/节奏问题。
5. **回填**：这条用了哪些套路、评分卡当初几分、**预测卡 vs 实际的 diff**。
6. **更新建议**：套路库加权/降权、预测器校准、评分卡维度调整。
7. **👤 人工卡点**：建议先给用户，**确认后才改套路库**。

## 铁律（防过拟合）
- **单条不下结论**：同类攒够 2-3 条再改库。
- 数据 24-72h 才稳，别发完就判；建议 24h/72h/7天 各取一次。
- 闭环两条腿：套路库（什么会爆）+ 预测器（爆到什么程度），每条都喂。
